1. OpenStack基本介绍

OpenStack是一个开源的IaaS实现，它由一些相互关联的子项目组成，主要包括计算、存储、网络。由于以Apache协议发布，自2010年项目成立以来，超过200个公司加入了OpenStack 项目，其中包括AT&T、AMD、Cisco、Dell、IBM、Intel、Red Hat等。目前参与OpenStack 项目的开发人员有17,000+，来自139个国家。

OpenStack兼容一部分AWS接口，同时为了提供更强大的功能，也提供OpenStack风格的接口RESTFul API）。和其他开源IaaS相比，架构上松耦合、高可扩展、分布式、纯Python实现，以及友好活跃的社区使其大受欢迎，每半年一次的开发峰会也吸引了来自全世界的开发者、供应商和客户。

OpenStack由几个主要的组件组合起来完成具体的工作。主要的组件包括：RabbitMQ, Keystone, Glance, Nova, Neutron, Dashboard, Cinder, Swift, 和Ceilometer等。相关介绍如下:

  • RabbitMQ: 消息代理，协调服务中的操作和状态信息
  • Keystone: 身份验证、服务规则和服务令牌，它实现了OpenStack的Identity API
  • Glance: 提供镜像服务，旨在发现，注册和交付虚拟机磁盘和镜像
  • Nova: 提供计算虚拟化服务，是OpenStack的核心，负责管理和创建虚拟机
  • Neutron: 提供网络虚拟化服务，是一个可拔插，可扩展，API驱动的服务
  • Dashboard: 提供了一个图形控制台服务，让用户方便地访问，使用和维护OpenStack中的资源
  • Cinder: 提供块存储服务，为OpenStack的虚拟机提供持久的块级存储设备。支持诸如Ceph，EMC等多种存储后端
  • Swift: 提供对象存储服务，是一个分布式，可扩展，多副本的存储系统

我们采用neutron网络三节点架构的OpenStack，分别是Controller Node (控制节点)、Network Node (网络节点)和Compute Node (计算节点)。

2. 基本环境要求

我们的四台服务器均为安装的是Debian 7.5 wheezy系统(若你采用的是Ubuntu 14.04以上的系统，如下的安装操作均适用)，内核版本为3.2.57-3。分别使其中一台作为Controller Node，Network Node，两台作为Compute Node.

2.1 基本网络环境

配置eth1网络接口作为管理网络接口，设置为静态ip，Controller Node，Network Node，Compute1 Node，Compute2 Node的eth1接口ip地址分别为: 192.168.0.11，192.168.0.21，192.168.0.31，192.168.0.41。随后编辑各台服务器的hostname分别为controller，network, compute1和compute2，并将其他三个节点的管理网络ip和对应的hostname添加到hosts文件中。

2.1.3 验证连通性

在Controller节点，ping外部网络的域名地址，然后ping Network节点的管理网络接口，最后ping Compute节点的管理网络接口，若都ping得通，则Controller节点网络配置没问题。

在Network节点，ping外部网络的域名地址，然后ping Controller节点的管理网络接口，接着ping Compute节点的管理网络接口，最后ping Compute节点的实例通道接口，若都ping得通，则Network节点网络配置没问题。

在Compute节点，ping外部网络的域名地址，然后ping Controller节点的管理网络接口，接着ping Network节点的管理网络接口，最后ping Network节点的实例通道接口，若都ping得通，则Compute节点网络配置没问题。

2.2 NTP

为了保证多台机器的同步，必须安装NTP。四台机器上均使用apt-get install ntp命令安装ntp包。

2.4 OpenStack包的安装

根据我们的系统Debian Wheezy，选择当前最新的OpenStack Icehouse稳定版。

首先，添加Debian Wheezy的Icehouse补丁包库，echo “deb http://archive.gplhost.com/debian icehouse-backports main” >>/etc/apt/sources.list；接着，添加Debian Wheezy OpenStack文件库: echo “deb http://archive.gplhost.com/debian icehouse main” >>/etc/apt/sources.list；更新文件库数据库并安装gplhost-archive-keyring: apt-get update && apt-get install gplhost-archive-keyring；最后，更新包数据库，升级系统: apt-get update && apt-get dist-upgrade，重启，使所有修改生效: reboot。

2.5 消息服务

OpenStack使用消息代理来协调服务中的操作和状态信息，OpenStack支持几种消息代理，包括RabbitMQq，Qpid和ZeroMQ等，其中最典型的是RabbitMQ。我们选择使用RabbitMQ作为消息代理，并安装在Controller节点。
首先，使用命令apt-get install rabbitmq-server安装RabbitMQ；接着，配置消息代理服务。RabbitMQ创建默认账号guest，修改密码为123456，相应的命令为

rabbitmqctl change_password guest 123456

2.6 使用debconf配置OpenStack

Debian 中的debconf和dbconfig-common两个包用来执行关于OpenStack中各个服务组件的配置操作。在安装OpenStack相关组件包时，debconf以一个交互式的页面来提示用户配置相关服务的配置文件。在这个过程中并不能把所要修改的内容修改完全，并且提示界面不能很好反映我们所配置的内容含义，所以我们选择直接在系统的etc目录下修改相关服务组件的配置文件，并使用下面的命令禁用debconf。

dpkg-reconfigure -plow debconf
